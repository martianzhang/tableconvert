Usage: tableconvert [OPTIONS] [INPUT_FILE] [OUTPUT_FILE]

ðŸš€ Convert between different table formats (MySQL, Markdown, CSV, JSON, Excel, etc.)

BASIC OPTIONS:
  --from|-f={FORMAT}        Source format (e.g. mysql, csv, json, xlsx)
  --to|-t={FORMAT}          Target format (e.g. mysql, csv, json, xlsx)
  --file|--input|-i={PATH}  Input file path (or stdin if not specified)
  --result|--output|-o={PATH} Output file path (or stdout if not specified)

QUICK OPTIONS:
  -v, --verbose             Enable verbose output for debugging
  -h, --help                Show this help message
  --version                 Show version information
  --help-formats            Show all supported formats and their parameters
  --help-format={FORMAT}    Show parameters for a specific format
  --mcp                     Run as MCP (Model Context Protocol) server

DATA TRANSFORMATIONS:
  --transpose               Transpose the table (swap rows and columns)
  --delete-empty            Remove empty rows from the table
  --deduplicate             Remove duplicate rows
  --uppercase               Convert all text to UPPERCASE
  --lowercase               Convert all text to lowercase
  --capitalize              Capitalize the first letter of each cell

BATCH PROCESSING:
  --batch|-b={PATTERN}      Process multiple files matching a pattern
                            Examples: "*.csv", "data/*.json", "**/*.xlsx"
  --recursive|-r            Enable recursive directory traversal
  --output-dir|--dir={PATH} Specify output directory (default: same as input)

FORMAT-SPECIFIC OPTIONS:
  Each format supports its own extension parameters. Examples:
    --align=l,c,r           For markdown: column alignment (left/center/right)
    --style=box             For ascii: table style (box, plus, dot, bubble)
    --delimiter=TAB         For csv: value delimiter (COMMA, TAB, SEMICOLON, etc.)
    --format=object         For json: output format (object, 2d, column, keyed)
    --minify=true           For html/json/xml: minify output
    --table=mytable         For sql: table name for INSERT statements
    --template=file.tmpl    For tmpl: template file path
    --bold-header           For markdown/latex: make headers bold
    --auto-width            For excel: auto-adjust column widths

EXAMPLES:
  # Basic conversion with auto-detection
  tableconvert input.csv output.json

  # Explicit format specification
  tableconvert --from=csv --to=markdown input.csv output.md

  # Short flags
  tableconvert -i input.csv -o output.json

  # Stdin to stdout
  echo "name,age\nAlice,30\nBob,25" | tableconvert --from=csv --to=json

  # With format-specific options
  tableconvert data.csv report.md --bold-header --align=l,c,r

  # Transform data
  tableconvert data.csv output.md --transpose --capitalize

  # Batch processing
  tableconvert --batch="data/*.csv" --to=json --output-dir=results

  # Batch with recursion
  tableconvert --batch="**/*.csv" --to=json --recursive --verbose

  # SQL generation
  tableconvert data.csv data.sql --table=users --dialect=mysql --one-insert

  # LaTeX for papers
  tableconvert data.csv table.tex --caption="Results" --text-align=c

  # Show format help
  tableconvert --help-format=markdown

  # Show all formats
  tableconvert --help-formats

  # Run as MCP server
  tableconvert --mcp

AUTO-DETECTION:
  When --from or --to are not specified, tableconvert will attempt to detect
  the format from file extensions:

    .csv      -> csv
    .json     -> json
    .jsonl    -> jsonl
    .md, .markdown -> markdown
    .xlsx, .xls -> excel
    .html, .htm -> html
    .xml      -> xml
    .sql      -> sql
    .tex, .latex -> latex
    .wiki     -> mediawiki
    .tmpl, .template -> tmpl
    .txt      -> (not auto-detected, must specify)

  Note: .txt files require explicit format specification with --from
